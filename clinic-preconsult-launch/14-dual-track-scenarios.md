# 프리체크인 듀얼 트랙 시나리오 정리

작성일: 2026-04-03

## 1. 지금 사실로 말할 수 있는 것
### 강한 가설
- 고객이 예약 전에 기준을 정리하지 못하면 사업자 응대 시간이 길어진다.
- 사업자는 상담 전 더 구조화된 정보를 원할 가능성이 높다.
- 사업자 링크 앞단에 pre-checkin을 붙이는 구조는 설명이 쉽다.

### 아직 검증 안 된 것
- 병원이 실제로 돈을 낼지
- 뷰티 사업자가 실제로 돈을 낼지
- 누구에게 더 빨리 "이거 바로 써보겠다" 반응이 나오는지
- 완료율과 CTA 클릭률이 의미 있게 나오는지

즉 현재 상태는 **pain hypothesis는 있음, willingness-to-pay는 아직 미검증**이다.

---

## 2. 공통 제품 코어
둘 다 같은 코어를 쓴다.

1. 사업자가 링크를 보낸다.
2. 고객이 사전 체크인을 작성한다.
3. 고객은 준비 요약을 본다.
4. 사업자는 요약 카드를 본다.
5. 파일럿 동안 시작 수 / 완료율 / CTA 클릭을 본다.

공통으로 풀려는 문제:
- 상담 전에 필요한 기준이 정리되지 않음
- 반복 설명이 길어짐
- 부적합 리드를 너무 늦게 파악함

## 3. 클리닉 트랙
### 돈 내는 사람
- 상담실장
- 운영팀
- 원장

### 고객 시나리오
- 카톡채널 또는 예약 링크 앞에서 사전 체크인 진입
- 관심 시술군 / 부위 / 예산 / 통증 민감도 / 다운타임 / 방문 시간 입력
- 상담 전에 질문과 우선순위를 정리한 상태로 들어감

### 사업자 시나리오
- 고객의 기준을 요약 카드로 먼저 확인
- 첫 3~5분 반복 설명 감소 여부 확인
- 부적합 문의를 조금 더 빨리 구분

### 조심해야 하는 것
- 시술 추천처럼 보이면 안 됨
- 병원 비교나 송객처럼 보이면 안 됨
- 반드시 "상담 전 정보정리"로 설명

### 핵심 검증 질문
- 상담실이 바로 공감하는 pain인가?
- 의료 오해 없이 설명 가능한가?
- 7일 테스트를 열어볼 만큼 가치가 있나?

---

## 4. 뷰티 사업자 트랙
### 돈 내는 사람
- 샵 운영자
- 실장
- 원장

### 고객 시나리오
- 샵 링크 앞단에서 사전 체크인 시작
- 목표 / 부위 / 예산 / 민감도 / 시간대 / 우려사항 입력
- 상담 전에 기대치를 더 정리한 상태로 문의

### 사업자 시나리오
- 카톡/DM 전에 고객 목적과 예산을 먼저 확인
- 예약 적합도와 우선순위를 빠르게 판단
- 상담 메시지 길이와 미스매치를 줄일 수 있는지 확인

### 기대 장점
- 의료 톤 없이 설명 가능
- "예약 전 기대치 정리"로 자연스럽게 설명 가능
- 규제 리스크가 상대적으로 낮음

### 핵심 검증 질문
- 샵 운영자가 바로 이해하는 pain인가?
- 도입 설명이 병원보다 쉬운가?
- 예약 전환 또는 응대 효율 개선 가능성이 보이는가?

---

## 5. 이번 소프트웨어 구현이 의미하는 것
이번 구현은 아직 최종 제품이 아니다.
대신 아래를 빠르게 비교할 수 있게 한다.

- 어느 vertical이 더 공감이 빠른지
- 어느 vertical이 완료율이 더 높은지
- 어느 vertical이 CTA 클릭이 더 잘 나오는지
- 어느 vertical이 실제 파일럿 대화로 더 쉽게 넘어가는지

즉 이 앱의 역할은
**"병원 vs 뷰티 중 어디가 더 빨리 wedge가 되는지 비교하는 실험기"**다.
